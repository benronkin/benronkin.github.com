Google Sheets to Google Cloud Storage

Google Cloud

  • Key highlights

  • Pull campaign data from Google Sheets
  • Group data by campaign ID
  • Create or update CSV for each campaign
  • Upload CSVs to Google Cloud storage
  • Set file permissions
  • Email CSV file download link

Summary

The project required developing a Node.js app on Google Cloud Platform that processes campaign registrations from Google Sheets. The app reads the sheet via Google Sheets API, groups the registrations by campaign ID, and creates a CSV file for each campaign. The file then gets uploaded to Google Cloud Storage, where the app sets access permission on the file. Finally, the app emails a download link to the CSV file.

Read Google Sheet

Campagin registrations are collected via a form and saved in a Google Sheet. The app contains a service account that is permitted to access the sheet.

The app reads the sheet periodically, being triggered by a Google Cloud Scheduler that publishes a message to a Google Pub/Sub topic.

Create Cloud Storage files

The app groups together all leads that belong to the same campaign. The leads are stored in a CSV file that the app saves to Google Cloud Storage.

Insert or update files

The app saves individual files to a Google Cloud Storage bucket. The app sets access permissions on each file.

Since Cloud Storage objects are immutable, the app uses a smart technique to update existing files with new leads.

Email CSV file links

A Google Apps Script scans the Cloud Storage bucket for new files. Whenever new files are encountered, the script emails a link to the CSV file.
