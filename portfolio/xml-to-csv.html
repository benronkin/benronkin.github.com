SUMMARY

This Google Apps Script looks for new XML files in a Google Drive folder. It takes in a customizable mapping of XML entries to CSV headers, and automatically converts the XML to a CSV file. Both input and output files are then automatically moved to different Google Drive folders for subsequent automation.

Customizable file location

Users can specify Google Drive folders where new XML files will be stored, where they should be archived, and where the CSV files should be stored.

Clients can also configure the name of the CSV file.

The conversion process can be initiated either manually via a custom menu option, or automatically via a timer.

Customizable field mapping

Users can modify the mapping of XML entries to CSV columns at any time without any code modifications. They simply enter a CSV header and map it to the nested XML element.

Users can also specify formatting requirements for each field; f.g, split full names to first and last, format dates, etc.

Automated process

Once the script initiates, it creates a CSV file with a header row containing the columns from the mapping sheet. It uses recursion to iterate through the nested XML document, pulling elements and their attributes, and creating new CSV rows. It then compares each attribute to the mapping list. If if finds a specified attribute then it adds its value to the right position in the row.

Finally, the script saves the CSV to the specified drive, and moves the XML file to an archive folder.
