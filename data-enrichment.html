Summary

As a solo developer on the project, I created a sophisticated workflow system that handles new file submission in Google Apps Script. The system cuts down on the effort to validate and enrich submitted files, while providing stakeholders with a clear picture regarding the status of every submission.

Watch for new files

A Google Apps Script watches a specified GDrive folder for new file submissions.

Every submission is copied and archived automatically. Submissions are logged for reporting and analysis.

Flexible validation schemes

Different file submissions served different business purposes that required files adhere to different set of rules.

My GAS solution offered a customizable set of validation rules for every type of file. I made it super easy to add new validation schemes and update the schemes periodically as a need arises.

Customizable file formatting

My Google Apps Script needed to enforce spreadsheet column uniformity while allowing for different spreadsheet types.

I've built the script in such a way that made it easy to support new formats, as well as change existing format, by easily adding instructions regarding adding and removing columns, renaming columns, and more.

Email notifications for owners

Every file submitted to my Google Apps Script has to be routed to the right owner once formatted.

The Google Apps Script uses a GMail service and a lookup Google Sheet to alert users when new files are ready for final processing.
