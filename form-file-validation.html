Summary

The project involved creating a Google Form that accepts file uploads. Upon every upload, a Google Apps Script was to validate the file against a set of validation rules that are configurable by managers. The script was to email the file submitter if errors occurred. The script was to log every submission and its validation results. Valid files are moved to a destination folder.

Easy to use

Form managers have access to a configuration panel where they can turn on or turn off specific validation rules. They can also configure things such as required columns and then base validations on these headers.

It's easy to add custom validations to the script.

File submission form

A simple Google Forms enables end users to submit files. When a file is submitted, a Google Apps script is triggered. The script performs the set of validation rules previously selected by the managers.

The script moves files that successfully pass validation to a predefined destination folder.

Submission log

The script logs every file submission and its validation status to a separate Google Sheets. Managers can track and report on the various submissions.

Email notifications

When the script encounters validation errors for a submitted file, it triggers an email message to the submitter's email address (captured during file submission). The email contains a link to the submitted file and explanation regarding the validation errors.
